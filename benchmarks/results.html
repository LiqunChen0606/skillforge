Key Findings

1. Raw PDF text is the cheapest — but it's lossy

Raw PDF text extraction (561K tokens, 89.8% savings) beats every other format on raw token count. But it produces flat, unstructured text — no headings, no sections, no tables, no semantic blocks. Fine for simple Q&A; unsuitable when the model needs to reason about document structure.

2. AIF LML Aggressive is the best structured format

At 980K tokens (82.2% savings), AIF LML Aggressive is only ~75% more tokens than raw PDF text, but carries full semantic structure: typed sections, headings, claims, callouts, code blocks, tables, and figures. This enables structured navigation and reasoning — not just flat-text scanning.

3. AIF beats raw Markdown by 5+ percentage points

Raw Markdown (77.1% savings) is good, but AIF LML Aggressive pushes that to 82.2%. The difference: AIF strips presentational noise while preserving — and typing — the semantics. Across 10 articles, that's 282,907 fewer tokens (1.26M → 0.98M).

4. LML modes converge for general documents

Unlike skills (where different prose modes show variation), all 5 LML modes produce nearly identical token counts for general documents (985K–980K, spread <0.6%). This is because general documents have fewer skill-specific blocks (steps, verification, etc.) where mode differences apply. For documents, just use Aggressive.

5. Context window multiplier effect

With a 200K context window, raw HTML fits ~36% of one Wikipedia article. AIF LML Aggressive fits ~20% per article — meaning you can load 5.6× more content in the same context. For RAG pipelines, this means 5.6× more relevant chunks per query.

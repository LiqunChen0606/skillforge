Key Metrics:
TNO (Token-Normalized Outcome) = Compliance ÷ (tokens / baseline_tokens). Measures semantic fidelity per token spent. TNO > 1.0 = better efficiency than baseline. TNO = 1.0 = equivalent.
Semantic Compliance = Fraction of semantic blocks (@step, @verify, @precondition, @skill) preserved in output. 100% = all structural elements survived conversion.

Key Findings

1. All text formats achieve 100% semantic compliance

Every AIF text format preserves every semantic block from the original skill. No @step, @verify, @precondition, or @skill block is lost. Format differences affect token count, not correctness.

2. Skills are already token-dense — savings are marginal

Unlike general documents (82% savings vs HTML), skills are already concise Markdown. The baseline averages 3,951 tokens per skill. The best format saves only 1.9% (751 tokens across 10 skills). Skills are authored for LLM consumption — there's little noise to strip.

3. Markdown RT wins for skills; LML Aggressive is a close second

Markdown (RT) delivers the highest TNO (1.05) with slightly fewer tokens. LML Aggressive (TNO: 0.99) is nearly identical but provides explicit typed tags — useful when the LLM needs to identify block types programmatically.

4. Small skills show highest variance

The smallest skills (debugging: 219 tokens, tdd: 321 tokens) show the widest swings: Markdown RT saves 14.6% on debugging but only 12.5% on tdd. Larger skills (>5K tokens) converge to <1% differences. Fixed per-format overhead dominates small documents.

5. Binary formats: 3.5× more tokens, 45% fewer bytes

Binary formats are 45% smaller in bytes than JSON IR (152K vs 278K) but base64 encoding inflates tokens to 179K. Use binary for storage and wire transport, never for LLM context.

Key Findings

1. Markdown roundtrip is the most token-efficient format

Markdown (RT) achieves +2.8% token savings (207.5K vs 213.4K), making it the cheapest text format. The roundtrip through AIF strips noise while preserving readable structure. This means you can fit 35 skills in a 200K context window vs 34 skills with raw SKILL.md.

2. LML Aggressive preserves full semantics at near-zero cost

LML Aggressive (209.7K tokens, +1.8% vs baseline) maintains 100% semantic compliance with typed block markers (@step, @verify, @precondition) at essentially the same token count as raw Markdown. Its TNO of +2.9% makes it the best choice when semantic structure matters for agent execution.

3. JSON IR costs 81% more tokens — avoid for LLM context

JSON IR (338.4K tokens) is -58.5% more expensive than the baseline. The verbose key-value syntax, escaping, and structural nesting add significant overhead. Reserve JSON for cross-language SDKs and machine parsing, not LLM consumption.

4. Skill size drives format efficiency

Small skills (219-321 tokens) see the largest variance: the "debugging" skill saves 14.6% with Markdown RT but loses 8.7% with LML Aggressive. Larger skills (5K-8K tokens) converge — overhead from format tags is amortized. This suggests LML Aggressive is optimal for production skills (>1K tokens), while Markdown RT wins for very compact skills.

Binary Formats: Wire Transport, Not LLM Context

Binary Wire and Binary Token formats are ~82% smaller in bytes than JSON IR (749.3K bytes vs 1245.5K bytes), making them ideal for storage and network transport.

However, when base64-encoded for LLM consumption, they inflate to 876.7K tokens — a -310.8% overhead vs baseline. Never feed binary formats directly to LLMs.

Methodology

Pipeline: Each SKILL.md file is imported via aif skill import --format <fmt>, producing output in 11 formats. Token counts are measured via Claude's count_tokens API (model: claude-opus-4-6), which returns exact BPE token counts — not heuristic estimates.

Compliance scoring: For each format, we count semantic block markers (@step, @verify, @precondition, @skill and their LML/HTML equivalents) and compare against the ground-truth count from the JSON IR AST. Compliance = matched / expected.

TNO (Token-Normalized Outcome): compliance / (format_tokens / baseline_tokens). A TNO of 1.0 means the format costs the same as baseline with full compliance. TNO > 1.0 means it's both cheaper AND fully compliant — strictly better. TNO < 1.0 means you're paying more per unit of semantic fidelity.

Fixtures: 36 production-quality skills from the Superpowers and Figma ecosystems, ranging from 219 to 7,795 baseline tokens. This covers small utility skills through large multi-step workflows.

Key Findings

1. Raw PDF text is the cheapest — but it's lossy

Raw PDF text extraction (561K tokens, 89.8% savings) beats every other format on raw token count. But it produces flat, unstructured text — no headings, no sections, no tables, no semantic blocks. The LLM receives a wall of text with no navigational cues. This is fine for simple Q&A over content, but unsuitable when you need the model to reason about document structure, follow cross-references, or process specific sections.

2. AIF LML Aggressive is the best structured format

At 980K tokens (82.2% savings vs raw HTML), AIF LML Aggressive is only ~75% more tokens than raw PDF text, but carries full semantic structure: typed sections, headings, claims, callouts, code blocks, tables, and figures. This enables the LLM to navigate, reference, and reason about the document as a structured object — not a flat string.

3. AIF beats raw Markdown by 5+ percentage points

Raw Markdown (77.1% savings) is good, but AIF LML Aggressive pushes that to 82.2%. The difference: AIF strips presentational noise while preserving — and typing — the semantics. Across 10 Wikipedia articles, that's 283K fewer tokens (1.26M → 0.98M).

5. Binary formats: great for wire, bad for LLM context

Binary wire and token-optimized formats are ~82% smaller in bytes than JSON IR, but base64 encoding inflates their token count to ~179K per article. Use binary for storage and transport between systems, not for feeding to LLMs.

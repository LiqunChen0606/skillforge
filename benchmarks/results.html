Key Insight: PDF files sent natively to Claude cost ~75% fewer tokens than raw HTML — on par with Markdown. Extracted PDF text (plain text, no formatting) achieves ~90% reduction but loses all structure. The AIF JSON IR is a verbose AST for programmatic use; the real token-efficient semantic formats are AIF source (31%) and LML (35%). Markdown wins on raw token count (77%) but loses semantic block types.

Format Trade-offs:
  • PDF (extracted text) — Best raw token efficiency (~90%) but loses all formatting, structure, and layout
  • PDF (native file) — Good reduction (~75%) when sent as a document, but opaque binary format
  • Markdown — Strong token efficiency (~77%) but loses semantic block types (definitions, theorems, callouts)
  • LML — Optimized for LLM consumption, preserves semantics with minimal syntax overhead
  • AIF Source — Full fidelity format with all metadata, attributes, and structure
  • AIF JSON IR — Verbose AST for programmatic processing, not intended for direct LLM consumption
